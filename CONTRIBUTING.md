# Contributing to Sortie

Thank you for your interest in improving Sortie! We welcome contributions from everyone.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please report any unacceptable behavior to <hello@rjsadow.com>.

## Getting Started

1. **Fork** the repository and clone it locally.
2. Create a new **branch** for your feature or bug fix: `git checkout -b feat/your-feature-name`.
3. Set up your environment (requires **Go 1.24+** and **Node.js 22+**).

## How to Contribute

### Reporting Issues
- Use the GitHub issue tracker to report bugs or suggest features.
- Provide a clear description and steps to reproduce for bugs.

### Pull Requests
- Ensure your code follows the existing style (Go for backend, TypeScript/React for frontend).
- Update the documentation if you introduce new features or configuration variables.
- Open a Pull Request (PR) against the `main` branch with a descriptive title.

## Development Commands

- Run `make dev` to start both the Go backend and React frontend.
- Use `docker build` to test containerization before submitting.

We appreciate your help in making Sortie a better tool for everyone!
